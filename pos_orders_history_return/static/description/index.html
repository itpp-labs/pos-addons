POS Orders Return

Make order returns from POS interface by quick & easy way

Key features:
  • All features from POS Orders History
  • Make order return process from POS
  • Track all the returned orders and their details

By default, Odoo has not a convenient way to make order return process. Our module eliminates this disadvantage and allows to quick & easy do a return transaction.

How It Works

Order Return Process

  • Open POS session and click History
  • Click Return for the paid order to be returned


As a result, the order is opened in Return Mode. Here you can see only the products belonging to current order with quantities not higher than paid originally. So you can't return more products than sold.


The quantity for return decreases, once you add the products to order

Finally, you can complete the return process by using usual payment transaction


No Receipt Returns

The option No Receipt is helpful if a customer, who demands the return, lost his payment receipt.


When we use No Receipt all products are available for return


Order Search

Note that you can easily find the orders to be returned by several ways:

  • Search Bar


  • Receipt Barcode

Scanning the barcode immediately selects the transaction and opens return mode


Configuration

Go to Point of Sale → Configuration → Point of Sale and open POS configuration form. See the following options:

  • Return Orders - to activate/deactivate order returns
  • Show Returned Orders - to display all previously returned orders
