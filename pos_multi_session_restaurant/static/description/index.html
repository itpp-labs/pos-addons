Sync POS orders across multiple sessions (restaurant extension)

E.g., create order for a table in restaurant POS1, expand it in POS2, get payment in POS3.

Usage

The module useful for canteens, which have a tables with waiters as well as usual queue.

Before using the module be sure that your odoo instance support longpolling, i.e. Instant Messaging works (the instructions for setting up can be found here).


Once all dependencies are set up, go to "Point of Sale → Configuration → Point of Sales" and create or open some restaurant Point of Sale record (not a queue POS).
Set Multi-session value (create new one if need) and select floor in "Restaurant floors" field (for more information about creating POS and setting Multi-session value follow the link. If a floor record has not yet been exist, you can create it in Point of Sale → Floor Plans). Click the "Save" button to save your changes. Set the same Multi-session and floor for other "Point of Sale", which should be synced.

Also you can attach all orders from other POSes to a so-called "Default table". It is used for orders that had not been attached to any of tables. The corresponding field is located in "Multi-session" section.

It should be noted that default table must be attached to any floor (do not foget to select this floor in "Restaurant floor" field), otherwise you get the error message "Uncaught Error: Default table is not belonged to this POS" after starting POS session.

Start POS session on all POSes:

  • login as user1 on computer1 ⇒ start session on POS "Rest1"
  • login as user2 on computer2 ⇒ start session on POS "Rest2"
  • etc

Next you need to create a tables in Rest1(user1) session.

The created tables will be displayed in Rest2(user2) respectively. Tables are synced according to floors and table id.

When you start to create order for Table1 (by clicking on the table) in Rest1(user1) session, you can see how items are displayed in the synced session Rest2(user2). Moreover, items added at the corresponding POS are marked as "added by Rest1", "added by Rest2" and etc.
